# 局域网临时 AI 会议室

一个基于 Python 的、可在局域网内使用的临时网络会议系统。主持人可在 Windows 或 macOS 设备上启动服务，局域网内成员通过浏览器加入会议，进行多语言聊天、AI 翻译、文件共享和会议纪要整理。

## 功能

- 局域网多人实时聊天（WebSocket）
- 翻译模式二选一：开启双向翻译 / 不翻译
- 支持接入 AI API 实现聊天内容同步互译
- 主持人手动生成 AI 会议纪要
- 纪要预览与下载（MD / TXT）
- 支持在会议过程中上传并共享文件
- 常用图片格式文件可直接在会议窗口中显示
- 其他类型文件可下载到本地后使用对应程序打开
- 修复了同一条消息可能被重复发送的 bug
- 解散会议后立即清空会议数据与临时文件
- 异常退出后下次启动自动清理遗留临时文件

## 适用场景

- 不同语言人员参与的局域网会议讨论
- 小型团队内部协作会议
- 需要实时翻译辅助的交流场景
- 需要在会议中临时共享文件的讨论活动
- 需要快速整理会议纪要的讨论活动

## 环境要求

- Python 3.10+
- Windows 或 macOS
- 同一局域网环境
- 可选：可用的 AI 服务商 API Key（用于翻译和 AI 纪要）

## 安装与配置

1. 打开项目目录。
2. 复制 `.env.example` 为 `.env`。
3. 在 `.env` 中填写所需配置项，例如 API Key、模型名称、服务地址等。
4. 安装依赖。

### Windows

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

### macOS

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

## 启动

### Windows

可直接双击：

```txt
run.bat
```

或在终端执行：

```bash
python -m app.main
```

### macOS

在项目目录终端中执行：

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
python3 -m app.main
```

启动后，主持人可通过本机浏览器访问：

```txt
http://127.0.0.1:8000/
```

## 局域网使用

1. 主持人页面会显示会议码、二维码、局域网地址。
2. 参会者在同一局域网下，用浏览器打开二维码链接或访问局域网地址。
3. 输入会议码、昵称、语言后加入会议。
4. 参会者可在聊天窗口中发送消息、查看翻译内容，并在需要时上传文件共享给会议成员。

## 文件共享说明

- 在聊天窗口边上的添加按钮可选择并上传文件。
- 常用图片格式文件可直接在会议窗口中显示。
- 其他文件类型通常以可下载形式提供，需下载到本地后使用对应软件打开。
- 请在可信局域网环境中使用文件共享功能，并注意文件内容安全。

## 主持人操作

- 修改会议名称与翻译模式
- 查看在线成员
- 点击“生成会议纪要”
- 下载纪要（MD / TXT）
- 点击“解散会议室”结束并清理数据

## 注意事项

- `.env` 文件不要上传到 GitHub。
- API Key 仅在服务端读取，前端不可见。
- 若未正确配置 AI 服务，翻译功能可能不可用，纪要功能可能回退到本地兜底逻辑。
- 文件上传与下载功能应仅在可信局域网环境中使用。
- 会议解散后，消息、翻译、纪要、成员会话与临时文件都会删除。
- 本项目默认面向可信局域网环境，不建议未经安全加固直接暴露到公网。

## 常见问题

### 参会者无法访问

请确认：

- 所有设备都在同一局域网内
- 防火墙允许 Python 进程通信
- 主持人设备的局域网 IP 地址可被其他设备访问

### 翻译失败

- 原文通常仍会保留
- 请检查 API Key、模型配置和网络连接
- 请确认所使用的 AI 服务商接口可正常调用

### 文件无法显示或打开

- 图片类文件请确认格式是否为浏览器支持的常用图片格式
- 其他类型文件通常需要下载后用本地程序打开
- 请检查文件是否上传成功以及网络连接是否正常

### 纪要失败

- 页面会返回错误提示，可重试
- 请检查 AI 配置是否正确
- 若 AI 不可用，可确认是否存在本地兜底逻辑
